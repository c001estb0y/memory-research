# 公司大脑：为什么大多数公司有数据却没有记忆

> 原文：[Company Brain: Why Most Companies Have Data But No Memory](https://nanothoughts.substack.com/p/company-brain-why-most-companies)
> 作者：Ashwin Gopinath · 2026年5月4日

---

任何组织中最困难的部分之一是制度摩擦。对话丢失上下文。会议产生模糊的后续行动。人们带着对所决定内容的不同版本离开。随着时间推移，团队不再共享同一个现实。这首先不是一个 Agent 问题，而是一个人类协调问题。即使每个人都聪明、目标一致、全力以赴，协调仍然很难。AI 让这个问题更加明显，因为它加快了工作推进的速度，而工作背后的共享上下文往往依然脆弱。对于创始人或 CEO 来说，这是保持"创始人模式"和通过摘要进行管理之间的区别。Paul Graham 将创始人模式描述为不同于把组织架构图的"子树"当作黑箱来对待的方式；在我看来，它是与公司真相的直接接触：客户痛点、产品权衡、未兑现的承诺，以及在成为指标之前的弱信号（[链接](https://www.paulgraham.com/foundermode.html)）。

我一年多前创立 [Sentra](http://sentra.app/) 的原因之一，是因为我在多个大型机构中经历了这种摩擦，并且自己也多次创业。我尝试用不同的名字来称呼这个想法：企业智能、企业通用智能、AI 幕僚长。名字不断变化，但问题始终没变：一个组织能否记住足够多的东西，从而连贯地推理和行动？在过去几天里，YC 呼吁建设"公司大脑"之后，人们纷纷给我发消息问：这不就是你在做的事情吗？答案是肯定的。只是我们一开始没这么叫而已。在一个想法有名字之前，要精确把握它很难，有时候简洁比准确传播得更快。

Sentra 不是过去这一年里我唯一看着成长的东西。我的女儿 Satakshi 在 Sentra 注册成立一周后出生，看着她学习让这个问题变得显而易见。她不是从战略文档开始的。没有模式、本体论或路线图。她从碎片开始：面孔、声音、手势、伸手和被抱起。某样东西掉了，某个人做出反应，世界就这样被存储了。起初，这是记忆。然后记忆变成了模型。她开始预期、测试，最终在不确定、犯错或惊讶时对自己的推理进行推理。

公司并没有太大不同。它们通过积累碎片来成长：会议、Slack 对话、邮件、客户电话、支持工单、路线图讨论、销售异议、投资者更新、代码审查和走廊谈话。

问题在于，公司积累碎片的速度远快于将其转化为记忆的速度。组织记忆研究者将记忆定义为来自组织历史的、可以影响当前决策的存储信息，而交互记忆（transactive memory）研究解释了为什么团队依赖"谁知道什么"，而不仅仅是写下来的东西（[OLK5 综述](https://warwick.ac.uk/fac/soc/wbs/conf/olkc/archive/olk5/papers/paper8.pdf)、[PubMed](https://pubmed.ncbi.nlm.nih.gov/18020808/)）。公司之所以能运转，是因为 Sarah 记得客户为什么需要 SSO，Ravi 记得新手引导为什么延迟了，创始人记得为什么某笔交易比仪表盘更重要。

这就是为什么 YC 的框架很重要。YC 将 AI 自动化的障碍描述为散落在人们头脑中、邮件、Slack 对话、工单和数据库中的领域知识。它做了一个有用的区分：这不是全公司搜索或文档上的聊天机器人，而是一张关于公司如何运转的活地图（[Y Combinator](https://www.ycombinator.com/rfs)）。我认为这基本正确，但这个说法需要更清晰的定义。公司大脑不是一样东西，因为大脑也不是一样东西。它记忆、关联、预测、反思和协调行动。公司大脑需要同样的分层结构，所以我的定义很简单：**公司大脑是一个关于组织如何记忆、推理和行动的活的、有权限控制的模型。**

这听起来很抽象，所以让我们把它具体化。第一层是**事实记忆**：跨会议、消息、邮件、文档、工单、CRM 笔记、代码提交、事件、仪表盘、客户电话和支持对话中发生了什么的记录。它需要来源追溯、权限控制、时间戳和事实锚定。大多数人从这里开始，因为它看起来像是最明显的问题。公司到处都有数据，所以本能反应是连接工具、索引文档，然后让 Agent 搜索所有内容。这很有用，但这也是为什么许多"公司大脑"的尝试最终悄悄变成了品牌更好的搜索产品。事实记忆可以告诉你客户要求了 SSO。它可能告诉你什么时候、谁在通话中、转录稿在哪里。但它可能不会告诉你为什么 SSO 很重要、考虑了哪些替代方案、谁反对了，或者做了什么权衡。公司不仅仅靠事实运转。它靠的是**被解读过的事实**。

第二层是**上下文图谱**，或推理层。这是事实变成公司模型的地方。一次客户电话连接到一个商机。商机连接到一个产品缺口，缺口连接到一个工程权衡，权衡连接到一个路线图决策，决策连接到战略。大多数系统把这些作为独立的制品存储。公司大脑需要保持它们之间的关系。这也是元认知应该存在的地方：对推理的推理。公司大脑应该知道什么时候证据薄弱、什么时候上下文过时、什么时候团队有冲突的假设、什么时候一个承诺没有负责人、什么时候 Agent 需要帮助。公司以奇怪的方式遗忘。它们不仅仅忘记事实；它们忘记了为什么某个事实重要、导致决策的论证过程、反事实假设、尝试过什么，以及谁持有后来被证明正确的异议。这就是为什么组织记忆一直不仅仅是存储；它是被用于决策的记忆（[Walsh and Ungson PDF](https://skat.ihmc.us/rid=1255442505000_1811726224_21686/OrganizationalMemory-Walsh.pdf)）。

第三层是**行动协调**。大脑不仅仅记忆和思考，它还协调行动。它决定什么时候行动、等待、寻求帮助、升级和停止。公司大脑也应该如此：它不应该仅仅回答问题，还应该帮助组织做下一件正确的事。这可能意味着起草一封跟进邮件因为上次通话产生了一个承诺，创建一个工单因为同一个投诉出现在多个支持对话中，警告 CEO 三个团队正在基于不一致的假设工作，或者告诉 Agent 一笔退款可以自动处理而一个定价例外需要审批。这不同于普通的自动化。自动化执行已知的工作流。公司大脑从上下文中协调行动。这很重要，因为公司正在试图在碎片化数据上构建 Agent，而麦肯锡认为 Agent AI 需要更强的数据基础、数据谱系、访问控制和治理才能规模化（[McKinsey](https://www.mckinsey.com/capabilities/mckinsey-technology/our-insights/building-the-foundations-for-agentic-ai-at-scale)）。

这就是当前的 Agent 对话碰到更深层次公司大脑问题的地方。给 Agent 访问工具的权限是有用的。给它们访问索引过的公司数据是有用的。但如果组织没有保留数据背后的推理，这些都是不够的。Agent 失败不仅仅是因为公司缺少数据。它们失败是因为公司缺少关于数据为什么意味着它所意味的东西的记忆。

缺失的基底是人类沟通。会议、消息和邮件是组织现实被创造的地方。一份路线图来自辩论、客户压力、技术限制、判断和权衡。一个 CRM 字段不会解释为什么一笔交易延迟了；通话记录会。一个工单不会解释为什么一个问题重要；升级过程会。

当人们谈论 Agent 好像公司已经是可读的时候，这一点就被忽略了。大多数公司知识并不是整整齐齐地放在文档里。它在人与人之间、在当下、在人们决定什么重要的时候被创造。等到它变成一个工单或 PRD 时，大部分"为什么"已经被压缩掉了。

这就是为什么会议记录比人们想象的更重要，但也是为什么仅仅会议记录可能作为一个品类还不够。当许多这类公司成立时，转录本身还是一个有意义的产品切入点。这正在快速改变。我不会惊讶如果在即将到来的 macOS 版本中，类似 Granola 的转录功能默认就可用了。当那发生时，会议记录公司面临的问题变得更加困难：如果转录和基本摘要是免费的，持久的产品是什么？Granola 谈到连续会议之间的文档空白，上下文在那里蒸发（[Granola](https://www.granola.ai/blog/meeting-notes-back-to-back-meetings-context)），Otter 描述会议作为可搜索的洞察和工作流（[Otter.ai](http://otter.ai/)），TechCrunch 指出会议记录工具已经在超越转录，进入全工作空间搜索和连接应用（[TechCrunch](https://techcrunch.com/2026/04/28/otters-new-feature-lets-users-search-across-their-enterprise-tools/)）。这种转变是有道理的，因为真正的奖品不是转录。真正的奖品是将人类交互转化为组织记忆。

这种转变是有道理的，因为转录不是目的地，摘要也不是目的地。真正的奖品是将人类交互转化为组织记忆——而不假装转录本本身包含了决策背后的判断、不确定性、分歧和反事实。

企业搜索公司正在从检索走向综合和 Agent。Glean 描述其知识图谱为跨100多个连接器的公司内容、人员和活动模型（[Glean](https://www.glean.com/resources/guides/glean-knowledge-graph)）。工作流公司正在走向 Agent 编排：Zapier Agents 跨数千个应用工作，具有触发器、动作和审批（[Zapier](https://zapier.com/blog/zapier-agents-guide/)），而 ServiceNow 描述其平台为统一 AI、数据、工作流和治理（[ServiceNow](https://www.servicenow.com/platform.html)）。Dust 正在构建了解你公司并做工作而不仅仅是查找东西的 Agent（[Dust](https://dust.tt/home/enterprise)）。

每个人都从不同的切入点向同一个中心移动。知识工具知道什么存在，会议工具知道说了什么，工作流工具知道如何行动，Agent 工具知道如何尝试任务。公司大脑位于交叉点，因为有用的问题不仅仅是"发生了什么？"而是：为什么发生了，接下来应该发生什么，谁有上下文，公司应该记住什么？

这就是困难的部分。公司大脑处于四件事的交汇点：

**事实记忆 + 人类沟通 + 上下文图谱与推理 + 受治理的行动 = 公司大脑**

如果其中一个缺失，你得到的东西有用但不完整。没有沟通的事实变成了可搜索的档案。没有结构的沟通变成了转录和摘要。没有来源追溯的推理变成了看似合理的猜测。没有上下文的行动变成了脆弱的自动化。公司大脑是整合点。

关于如何构建它，仍然有一个开放性问题。一条路径是**聚合**。公司大脑连接到公司已经使用的工具：邮件、日历、Slack、文档、CRM、项目管理、支持、代码和工作流。这可能是大公司起步的方式，因为它们的上下文已经是分散的；麦肯锡也做了类似的区分，即渐进式集成和更全面的 Agent 转型之间的区别（[McKinsey](https://www.mckinsey.com/capabilities/mckinsey-technology/our-insights/rethinking-enterprise-architecture-for-the-agentic-era)）。

另一条路径是**垂直整合**。一家年轻公司从一开始就将记忆、推理和行动作为其操作系统的一部分。会议、决策、承诺和 Agent 行动在知识碎片化之前就在一个基底中被捕获。我不知道哪种架构会胜出，但越早开始的公司会有优势。

我一直在回想的一个问题是：这是为谁服务的？它不能只为领导层服务。如果公司大脑只是一个高管仪表盘，它就变成了有更好用户体验的监控。它也不能只为个人服务。如果它只是一个个人助手，它就不会成为组织记忆。

答案，我认为，是公司大脑通过在正确的抽象层次上为每个角色服务来为组织服务。对于一线贡献者，它回答：我需要什么上下文？为什么做了这个决定？尝试过什么？谁负责下一步？我即将影响哪个客户承诺？

对于经理，它回答：哪些承诺有风险，哪些决策被阻塞了，哪些假设冲突，哪些跟进从未变成工作？对于 CEO，它回答：公司在哪里偏离了方向，客户在说什么，哪些决策证据薄弱，公司知道什么而领导层还不知道？对于 Agent，它回答：我可以安全地做什么，我必须使用什么上下文，什么时候应该问人类？

这就是为什么培育一个公司大脑比改造一个更容易。在一家老公司里，上下文已经碎片化了。决策是几年前做出的。知道原因的人已经离开了。文档相互矛盾。仪表盘很干净，但记忆消失了。

在一家年轻公司里，大脑可以随着公司的形成而形成。每次会议、决策、客户信号、承诺和 Agent 行动都可以从一开始就成为记忆。公司不需要以后"实施 AI"。它可以在记忆、推理和行动作为原语的环境中成长。

这就是我在 Sentra 努力的方向。不是公司文档上的聊天机器人，不是另一个仪表盘，不仅仅是会议记录，也不仅仅是 Agent。机会在于构建公司的记忆基底：一个捕获事实、保留人类上下文、重建推理、协调行动的系统。我在别处写过将此称为[系统3思维](https://nanothoughts.substack.com/p/system-3-collective-intelligence?r=c9ls5)：超越个人推理的认知，在[群体和机构的层面](https://nanothoughts.substack.com/p/enterprise-general-intelligence-the?r=c9ls5)。真正成为 AI 原生的公司不会是那些在分散数据上嫁接 Agent 的公司。它们会是那些记得为什么自己的工作意味着它所意味的东西的公司。
