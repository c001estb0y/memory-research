MemOS / Local Agent Memory

本地记忆插件如何工作

基于公众号发布介绍和本地源码重新梳理：MemOS Local Plugin 2.0 要补的是通用大模型与真实用户之间的“最后一公里”，把 Agent 的每次执行加工成可审计、可归因、可复用的学习资产。

源码位置 D:\GitHub\memory-research\MemOS\apps\memos-local-plugin

它不是 Claude Code Hook 插件，而是给 Agent Host 使用的本地记忆插件。

关系定位 公众号主线 安装目录 OpenClaw 接入 单轮流程 记忆层级 三层检索 评分到 Skill 对比 claude-mem 如何验证
SECTION 01

MemOS 是体系，插件是本地落地形态

memos-local-plugin 是 MemOS 仓库中的一个应用包。它把 Reflect2Evolve 记忆算法封装成 Agent Host 可以加载的插件，让 Agent 具备跨任务记忆、经验归纳和技能结晶能力。

它的中心不是 UI，也不是安装脚本，而是 core/ 中的 MemoryCore。 Host 差异由 adapters/ 吸收，稳定协议放在 agent-contract/。

SECTION 02

公众号介绍的主线：执行即学习

文章的核心判断是：缺少的不是知识，而是一套让 Agent 在执行过程中把每一步都学进去的机制。

大模型负责通用推理，MemOS 负责学习你的本地世界。
最后一公里

你的代码库、目录结构、命名习惯、历史踩坑，预训练学不到，微调来不及，普通 RAG 又难以沉淀反馈。

执行链成为资产

动作、观察、反思、结果不再只是临时上下文，而是进入捕获、评分、归纳、抽象、复用流水线。

双层反馈回流

环境反馈判断“跑没跑通”，用户反馈判断“是不是符合期待”，两者一起影响历史步骤的价值权重。

多 Agent 共用核心

Hermes Agent 与 OpenClaw 通过轻薄 adapter 接入同一份核心，新的 Agent 也可以复用底层进化记忆引擎。

SECTION 03

安装时把代码和记忆数据分开

Windows 本地源码安装

cd D:\GitHub\memory-research\MemOS\apps\memos-local-plugin
npm install
npm run build:package
.\install.ps1 openclaw

公众号给出的快速安装

curl -fsSL https://raw.githubusercontent.com/MemTensor/MemOS/main/apps/memos-local-plugin/install.sh | bash

这条命令适合 macOS / Linux 用户从发布包快速安装；Windows 源码研究仍建议使用 install.ps1。

OpenClaw 运行目录

~\.openclaw\plugins\memos-local-plugin\   # 插件代码
~\.openclaw\memos-plugin\config.yaml      # 配置
~\.openclaw\memos-plugin\data\memos.db    # SQLite 记忆库
~\.openclaw\memos-plugin\skills\          # 结晶技能
~\.openclaw\memos-plugin\logs\            # 日志
SECTION 04

OpenClaw 适配器在进程内直接接入 MemoryCore

OpenClaw 加载插件后进入 adapters/openclaw/index.ts。 这个入口会初始化 core、注册工具、绑定 hook，并启动本地 viewer。

Viewer 默认地址：
http://127.0.0.1:18799
bootstrapMemoryCoreFull

加载 ~/.openclaw/memos-plugin/config.yaml，初始化存储、模型、pipeline。

registerOpenClawTools

注册 memory_search、memory_get、memory_timeline、skill_get 等工具。

createOpenClawBridge

把 before_prompt_build、agent_end、工具结果等 Host 事件映射到 core。

startHttpServer

启动本地 HTTP / SSE viewer，展示 traces、policies、world model、skills 和日志。

SECTION 05

每轮对话是一次检索、执行、沉淀、提炼循环

公众号用 Alpine 容器安装 Python 包失败举例：系统不只记“后来修好了”，而是记录每一步尝试、错误、观察和反思，让下次遇到类似环境问题时不用重新试一遍。

before_prompt_build：先召回

adapter 提取当前 turn，core 执行 Tier 1 Skill、Tier 2 Trace/Episode、Tier 3 World Model 检索，并把结果渲染成注入上下文。

Agent 执行任务

模型可以主动调用 memory 工具，也可以调用普通工具。普通工具结果会被记录为 outcome，但不会在当前 LLM step 中途静默注入新上下文。

agent_end：再沉淀

adapter 收集 assistant 回复、工具调用和结果，core 关闭 episode 并触发 L1 trace 捕获。

异步提炼

reward 对 trace 回传价值，L2 从高价值重复模式中归纳 policy，L3 压缩环境认知，Skill 模块把稳定策略结晶成可调用能力。

SECTION 06

四层记忆不是摘要堆积，而是经验演化

L1 Trace

具体经历

记录动作、观察、反思和值，作为所有高层记忆的事实地基。

L2 Policy

跨任务策略

从多个相似任务中归纳“有效做法”，例如测试模式、存储实现模式。

L3 World

环境认知

压缩项目结构、限制、领域约定，让 Agent 理解“这个项目通常怎么做”。

Skill

可调用能力

把高支持度、高收益的策略包装成 Skill，在未来任务中直接调用。

SECTION 07

三层按需检索让上下文精打细算

层级 召回内容 适合时机 价值
Tier 1 Skill 新任务入口，命中稳定技能 直接给 procedure、scope、避坑规则，形成任务骨架
Tier 2 Trace / Episode 执行受阻或遇到 edge case 补“上次具体怎么解决”的历史步骤
Tier 3 World Model 同主题、同项目、同环境任务 给项目背景、目录约定、领域画像，跳过重复探索
SECTION 08

有用性不是靠直觉，而是一路打分筛出来

记录链路：先把执行变成可评分证据

Adapter 把用户输入、assistant 输出、thinking、工具调用和工具结果统一成 turn / trace。 L1 trace 先保存“发生了什么”，随后 capture 会为每步补 reflection，并用 alpha 衡量这条反思是否忠实、具体、可迁移。

任务结束后，reward 模块用 task summary 与用户反馈得到 R_human。 这里的 human 不等于每次都要人手工打分，而是“人的目标是否被满足”：显式反馈会进入评分， 没有显式反馈时则由 LLM rubric 和保守启发式估计。

召回链路：优先召回已经证明有用的东西

R_human 会沿 episode 反向传播成每条 trace 的 value 和 priority。priority 越高，越容易在 Tier 2 被召回；负价值 trace 不会删除， 但更适合作为 decision repair 的反例。

当多个高价值 trace 形成重复模式，L2 才会归纳 policy；当 policy 的 support 和 gain 达标，才可能进一步结晶为 Skill。

层级 关键分数 谁来打分 影响
L1 Trace alpha、value、priority LLM judge + reward backprop 自动计算 决定哪些历史步骤值得召回，哪些只作为反例保留
L2 Policy support、gain、status 证据计数 + 对照价值计算 决定策略是 candidate、active 还是 archived
Skill eta、trialsPassed、用户正负反馈 自动试用结果 + 人类显式反馈混合 决定 Skill 是否从 candidate 晋升 active，或因低可靠度归档
简单说：不是纯 human 打分，也不是 agent 自己说了算。用户反馈是强信号； 没有用户反馈时，系统用 LLM rubric、公式回传、支持度、收益和试用结果自动估计有用性。
SECTION 09

它和 claude-mem 的侧重点不同

维度 claude-mem memos-local-plugin
接入对象 Claude Code / Cursor / Gemini 等 Hook 型环境 OpenClaw / Hermes 这类 Agent Host
注入时机 SessionStart 注入最近 observation timeline Turn start 执行三层检索后注入 InjectionPacket
记忆形态 Observation + Session Summary 为主 L1 Trace + L2 Policy + L3 World Model + Skill
核心价值 让编码助手跨会话记住项目历史 让 Agent 从重复任务中沉淀策略并改变未来行为
SECTION 10

验证链路是否跑通

1. 看目录

确认 ~\.openclaw\plugins\memos-local-plugin 和 ~\.openclaw\memos-plugin 已生成。

2. 开 viewer

访问 http://127.0.0.1:18799，查看 Overview、Traces、Retrieval、Logs。

3. 做连续任务

用同一个小项目做 6 到 10 轮相似开发任务，观察 L1、L2、L3、Skill 是否逐步出现。
